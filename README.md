## Engineering approach

I take ownership from an unclear requirement to a verified working result. I prefer small, observable components, explicit boundaries, defensive error handling, and repeatable tests. I use AI coding tools selectively to shorten research, boilerplate, test, documentation, and refactoring time; architecture, review, security decisions, and final verification remain my responsibility.

Public repositories are sanitized portfolio editions. Production credentials, customer data, private services, paid-source modules, and commercially sensitive integrations are intentionally excluded.
